Posted by John Doe on 2017-01-16

下滑这里查看更多内容

Hexo是一个快速，简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，
在几秒内，即可利用靓丽的主题生成静态网页。我的博客搭建史走了将近一个月从开始的Wordpress 到后来的Ghost 最后终于选择
了Hexo。

Hexo是什么

  • Hexo的中文官网
  • 作者Tommy Chen

我为什么要用Hexo

  • 原因很简单，用过一段时间的其他博客，总结出来一句话: 太烂了！

  • Tommy Chen：起源

  • 最重要的一点就是我有IntelliJ IDEA 基础，并且我只想单纯的搭建一个技术博客

环境

  • 系统 ：Windows 10 (64bit)

    由于我最近入手了一台 MBP ，所以Mac下部署Hexo本文同样适用

  • Git : 2.11.0

  • Node.js : 7.4.0

部署开始

Git安装

使用 git --version 命令验证 ，输出 : git version 2.11.0.windows.3 则安装成功

Node.js安装

  • 下载Node.js

  • 下载完成后双击运行 .msi 安装文件 , 全部下一步
    安装完成后 打开Bash , 使用 npm -v 命令验证 ,

    输出 4.0.5 则安装成功

Node.js 源设置

  • 由于 Node.js 的官方默认源是在国外 , 抽风的概率很高 , 好在淘宝提供了这个源 淘宝node.js

使用定制的 cnpm 命令行工具代替默认的 npm :

npmnpm install -g cnpm –registry=https://registry.npm.taobao.org

再执行这一整句

执行完成后验证一下是否可以使用淘宝的 cnpm 命令 : cnpm info express

安装 Hexo 框架

由于用了淘宝的 cnpm 代替了 npm 所以执行安装命令 :

cnpm install -g hexo-cli

这里安装时间不一定快 , 安装完有 WARN 警告也没关系

创建 Hexo 项目

在想要安装 Hexo 的位置新建 git_work_space 文件夹 , 比如我自己创建的目录

在 E:\git_work_space 目录下，那么就在这个目录下再新建一个 hexo 文件夹

进入该目录：cd e:/git_work_space/hexo

然后执行： hexo init , 这个时间可能也会很长有显示WARN也不要管

最后执行： cnpm install , 有显示WARN也不用管

启动 hexo 本地服务： hexo server

再浏览器里访问 ：http://localhost:4000/ 如果能访问并且是 hexo 的欢迎页面说明开启成功

再转到 Git Bash 中 , 按下 ctrl + c停止服务
